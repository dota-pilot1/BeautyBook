# 8. 프론트 — 회원가입 페이지

## 목표
`/register` 라우트에 폼. react-hook-form + zodResolver + authApi 연동. 서버 `fieldErrors` 를 폼 필드별 에러로 매핑.

## 파일
```
src/
├─ app/
│  └─ register/
│     └─ page.tsx
└─ features/
   └─ auth/
      └─ signup/
         └─ SignupForm.tsx
```

### `src/features/auth/signup/SignupForm.tsx` (client component)
핵심 로직:
```tsx
"use client";

import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { useRouter } from "next/navigation";
import { toast } from "sonner";
import { signupSchema, type SignupFormValues } from "@/shared/lib/validation/auth.schema";
import { authApi } from "@/entities/user/api/authApi";
import { getApiError, getFieldErrors } from "@/shared/api/errors";

export function SignupForm() {
  const router = useRouter();
  const {
    register,
    handleSubmit,
    setError,
    formState: { errors, isSubmitting },
  } = useForm<SignupFormValues>({
    resolver: zodResolver(signupSchema),
    mode: "onBlur",
  });

  const onSubmit = handleSubmit(async (values) => {
    try {
      // passwordConfirm 은 서버로 안 보냄
      const { passwordConfirm, ...payload } = values;
      await authApi.signup(payload);
      toast.success("회원가입이 완료되었습니다. 로그인해주세요.");
      router.push("/login");
    } catch (e) {
      const api = getApiError(e);
      const fields = getFieldErrors(e);

      // 서버 fieldErrors → RHF 에 매핑
      (Object.keys(fields) as Array<keyof SignupFormValues>).forEach((k) => {
        setError(k, { type: "server", message: fields[k as string] });
      });

      // 중복 이메일 같은 BusinessException (fieldErrors 없음)
      if (api && Object.keys(fields).length === 0) {
        if (api.code === "AUTH_001") {
          setError("email", { type: "server", message: api.message });
        } else {
          toast.error(api.message);
        }
      } else if (!api) {
        toast.error("요청 중 오류가 발생했습니다.");
      }
    }
  });

  return (
    <form onSubmit={onSubmit} className="space-y-4" noValidate>
      <Field label="이메일" error={errors.email?.message}>
        <input
          type="email"
          autoComplete="email"
          {...register("email")}
          className="w-full rounded-md border border-input bg-background px-3 py-2 text-sm"
        />
      </Field>

      <Field label="사용자명" error={errors.username?.message}>
        <input
          type="text"
          autoComplete="name"
          {...register("username")}
          className="w-full rounded-md border border-input bg-background px-3 py-2 text-sm"
        />
      </Field>

      <Field label="비밀번호" error={errors.password?.message}>
        <input
          type="password"
          autoComplete="new-password"
          {...register("password")}
          className="w-full rounded-md border border-input bg-background px-3 py-2 text-sm"
        />
      </Field>

      <Field label="비밀번호 확인" error={errors.passwordConfirm?.message}>
        <input
          type="password"
          autoComplete="new-password"
          {...register("passwordConfirm")}
          className="w-full rounded-md border border-input bg-background px-3 py-2 text-sm"
        />
      </Field>

      <button
        type="submit"
        disabled={isSubmitting}
        className="w-full rounded-md bg-primary text-primary-foreground py-2 text-sm font-medium disabled:opacity-60"
      >
        {isSubmitting ? "가입 중..." : "가입하기"}
      </button>
    </form>
  );
}

function Field({ label, error, children }: {
  label: string;
  error?: string;
  children: React.ReactNode;
}) {
  return (
    <label className="block space-y-1">
      <span className="text-sm font-medium">{label}</span>
      {children}
      {error && <span className="text-xs text-destructive">{error}</span>}
    </label>
  );
}
```

## 부가사항

### (선택) 이메일 중복 debounce 체크
폼 제출 전에 미리 알려주려면 `email` 필드 blur 시 `authApi.checkEmail(email)` 호출. **기본 스코프에서는 생략**. 서버에서 409 내려주는 걸로 충분.

### 로그인 페이지 안내 링크
가입 성공 후 `/login` 으로 라우팅. `/login` 페이지는 "로그인 구현" 문서에서 생성.

## 검증
1. `npm run dev`
2. `http://localhost:4100/register` 접속
3. 케이스별 테스트:
   - 빈 제출 → 필드별 zod 에러
   - 짧은 비번 → "영문/숫자/특수문자 포함 8자 이상" 메시지
   - 비번 불일치 → "비밀번호가 일치하지 않습니다"
   - 정상 제출 → toast + `/login` 리다이렉트
   - 중복 이메일 → email 필드에 "이미 사용 중인 이메일입니다"

## 주의
- `next/navigation` 의 `useRouter` 는 **App Router** 전용 (`"use client"` 필수)
- `noValidate` 로 브라우저 기본 검증 끄기 (zod 로 통일)
- Tailwind 클래스는 shadcn 스타일 토큰 (`bg-background`, `text-primary-foreground` 등) 사용 중 — `globals.css` 에 이미 정의됨
- shadcn 컴포넌트 (`<Input/>`, `<Button/>`, `<Label/>`) 를 나중에 `npx shadcn@latest add input button label` 로 추가하면 Field 래퍼 리팩터링 가능. 지금은 의존 최소화.
