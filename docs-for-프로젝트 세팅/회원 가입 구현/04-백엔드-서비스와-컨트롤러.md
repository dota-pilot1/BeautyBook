# 4. 백엔드 — Service & Controller

## 목표
`AuthService.signup(...)` + `AuthController` 로 `POST /api/auth/signup`, `GET /api/auth/check-email` 구현.

## 패키지
```
com.cj.twilio.callcenter.user
├─ application
│  └─ AuthService.java
└─ presentation
   └─ AuthController.java
```

## 파일

### `user/application/AuthService.java`
```java
package com.cj.twilio.callcenter.user.application;

import com.cj.twilio.callcenter.common.exception.DuplicateEmailException;
import com.cj.twilio.callcenter.user.domain.User;
import com.cj.twilio.callcenter.user.infrastructure.UserRepository;
import com.cj.twilio.callcenter.user.presentation.dto.SignupRequest;
import com.cj.twilio.callcenter.user.presentation.dto.SignupResponse;
import lombok.RequiredArgsConstructor;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
public class AuthService {

    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;

    @Transactional
    public SignupResponse signup(SignupRequest req) {
        if (userRepository.existsByEmail(req.email())) {
            throw new DuplicateEmailException();
        }
        String hash = passwordEncoder.encode(req.password());
        User saved = userRepository.save(
            User.createNewUser(req.email(), hash, req.username())
        );
        return SignupResponse.from(saved);
    }

    @Transactional(readOnly = true)
    public boolean isEmailAvailable(String email) {
        return !userRepository.existsByEmail(email);
    }
}
```

### `user/presentation/AuthController.java`
```java
package com.cj.twilio.callcenter.user.presentation;

import com.cj.twilio.callcenter.user.application.AuthService;
import com.cj.twilio.callcenter.user.presentation.dto.*;
import jakarta.validation.Valid;
import jakarta.validation.constraints.Email;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
@Validated
public class AuthController {

    private final AuthService authService;

    @PostMapping("/signup")
    public ResponseEntity<SignupResponse> signup(@Valid @RequestBody SignupRequest req) {
        SignupResponse res = authService.signup(req);
        return ResponseEntity.status(HttpStatus.CREATED).body(res);
    }

    @GetMapping("/check-email")
    public ResponseEntity<CheckEmailResponse> checkEmail(
        @RequestParam @Email(message="올바른 이메일 형식이 아닙니다.") String email
    ) {
        return ResponseEntity.ok(new CheckEmailResponse(authService.isEmailAvailable(email)));
    }
}
```

### `PasswordEncoder` 빈
시큐리티 설정에서 생성. 05 단계에서 함께 다룸. 지금 당장 필요한 것만 미리 말하면:
```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();  // 기본 strength=10
}
```

## 엔드포인트 스펙

| Method | Path | Body/Query | Success | Failure |
| --- | --- | --- | --- | --- |
| POST | `/api/auth/signup` | `SignupRequest` | 201 + `SignupResponse` | 400(검증), 409(중복) |
| GET  | `/api/auth/check-email?email=foo@bar.com` | query `email` | 200 + `{available: true/false}` | 400(형식) |

## 검증 (수동, 아직 시큐리티 permitAll 안 열면 401 뜸 → 05 이후 재확인)
```bash
# 회원가입
curl -i -X POST http://localhost:4101/api/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"test@example.com","password":"Test123!@","username":"홍길동"}'

# 중복 재시도 → 409
curl -i -X POST http://localhost:4101/api/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"test@example.com","password":"Test123!@","username":"홍길동"}'

# 검증 실패 (짧은 비번) → 400 + fieldErrors.password
curl -i -X POST http://localhost:4101/api/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"bad@example.com","password":"short","username":"짧"}'

# 이메일 중복 체크
curl -i 'http://localhost:4101/api/auth/check-email?email=test@example.com'
```

## 주의
- `@Transactional` 은 `application` 레이어에만. 컨트롤러엔 금지.
- 컨트롤러에서 엔티티를 **절대** 반환하지 않음 — 항상 DTO.
- `@Validated` 클래스 애노테이션은 `@RequestParam` 파라미터의 `@Email` 검증을 트리거하기 위한 것.
