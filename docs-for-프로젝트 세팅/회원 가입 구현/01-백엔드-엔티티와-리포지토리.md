# 1. 백엔드 — 엔티티 & 리포지토리

## 목표
`User` JPA 엔티티와 `UserRepository` 만들기. 이후 단계의 **빌딩 블록**.

## 패키지 구조 (확정)
```
com.cj.twilio.callcenter
├─ TwilioCallcenterServerApplication.java  (기존)
├─ user
│  ├─ domain
│  │  ├─ User.java
│  │  └─ UserRole.java
│  └─ infrastructure
│     └─ UserRepository.java
```

> 참고: 원본 mapo는 `user/domain`, `user/application`, `user/presentation`, `user/infrastructure` 4분할. 우리도 동일하게 가되, 회원가입 단계에서는 domain/infrastructure 만 먼저.

## 생성할 파일

### `user/domain/UserRole.java`
```java
package com.cj.twilio.callcenter.user.domain;

public enum UserRole {
    ROLE_USER,
    ROLE_ADMIN
}
```

### `user/domain/User.java`
- 테이블명: `users` (PostgreSQL 예약어 `user` 피하기)
- 필드:
  | 필드 | 타입 | 제약 |
  | --- | --- | --- |
  | `id` | `Long` | `@Id @GeneratedValue(IDENTITY)` |
  | `email` | `String` | `unique=true, nullable=false, length=255` |
  | `passwordHash` | `String` | `nullable=false, length=100` (BCrypt 60자 + 여유) |
  | `username` | `String` | `nullable=false, length=50` |
  | `role` | `UserRole` | `@Enumerated(STRING), nullable=false, length=20` |
  | `active` | `boolean` | `nullable=false, default true` |
  | `createdAt` | `Instant` | `@CreationTimestamp, nullable=false, updatable=false` |
  | `updatedAt` | `Instant` | `@UpdateTimestamp, nullable=false` |

- Lombok: `@Getter`, `@NoArgsConstructor(access=PROTECTED)` (JPA용), `@Entity`, `@Table(name="users")`
- **세터 금지**. 변경은 도메인 메서드로.
- 팩토리 메서드:
  ```java
  public static User createNewUser(String email, String passwordHash, String username) {
      User u = new User();
      u.email = email;
      u.passwordHash = passwordHash;
      u.username = username;
      u.role = UserRole.ROLE_USER;
      u.active = true;
      return u;
  }
  ```
- 도메인 메서드 (나중 로그인 단계에서 쓸 것, 지금 미리 넣어둠):
  - `boolean isActive()` — Lombok `@Getter` 가 만들어줌
  - `void deactivate()` / `void activate()`

### `user/infrastructure/UserRepository.java`
```java
package com.cj.twilio.callcenter.user.infrastructure;

import com.cj.twilio.callcenter.user.domain.User;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;

public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);
    boolean existsByEmail(String email);
}
```

## 검증 방법
1. `./gradlew build` — 컴파일 성공
2. 앱 재기동 → Hibernate 로그에 `create table users` 보여야 함 (`ddl-auto: update`)
3. `docker exec -it twilio-callcenter-postgres psql -U postgres -d twilio_callcenter -c '\d users'` — 스키마 확인
   - 컬럼: `id, email, password_hash, username, role, active, created_at, updated_at`
   - `email` UNIQUE 제약 있어야 함

## 주의
- `passwordHash` 필드명 중요. `password` 라고 지으면 잘못 노출될 위험.
- `@JsonIgnore` 같은 건 DTO 로 분리하므로 엔티티에 달지 않음.
- 인덱스: `email` 은 unique 제약으로 자동 인덱스 생김. 추가 인덱스 불필요.
