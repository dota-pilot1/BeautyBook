# 5. 백엔드 — Security 설정 (회원가입 범위만)

## 목표
Spring Security 기본 인증(Basic Auth, 랜덤 패스워드)을 **끄고**, `PasswordEncoder` 빈 등록 + `/api/auth/**` 만 permitAll. JWT 필터는 **로그인 구현 단계에서** 추가.

## 패키지
```
com.cj.twilio.callcenter.config
└─ SecurityConfig.java
```

## 파일

### `config/SecurityConfig.java`
```java
package com.cj.twilio.callcenter.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configurers.AbstractHttpConfigurer;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;
import org.springframework.web.cors.CorsConfiguration;
import org.springframework.web.cors.CorsConfigurationSource;
import org.springframework.web.cors.UrlBasedCorsConfigurationSource;

import java.util.List;

@Configuration
public class SecurityConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(
                    "/api/auth/signup",
                    "/api/auth/check-email"
                ).permitAll()
                // 로그인 구현 단계에서 /api/auth/login, /api/auth/refresh 추가
                .anyRequest().authenticated()
            )
            .httpBasic(AbstractHttpConfigurer::disable)
            .formLogin(AbstractHttpConfigurer::disable);
        return http.build();
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowedOrigins(List.of("http://localhost:4100"));
        config.setAllowedMethods(List.of("GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS"));
        config.setAllowedHeaders(List.of("*"));
        config.setExposedHeaders(List.of("Authorization"));
        config.setAllowCredentials(true);
        config.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        return source;
    }
}
```

## 검증
```bash
# 1) 기동 로그에 "Using generated security password" 메시지가 더 이상 안 나와야 함
tail -f /tmp/twilio-back.log | grep -i "generated security password"

# 2) /api/auth/signup 은 익명으로 접근 → 201
curl -i -X POST http://localhost:4101/api/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"a@a.com","password":"Test123!@","username":"가나"}'

# 3) 미보호 경로 아닌 곳 → 401 (현재는 로그인 미구현이라 정상)
curl -i http://localhost:4101/api/some-other-endpoint
```

## 원본(mapo) 과 다른 점 — 왜?
| 항목 | mapo | 우리 | 이유 |
| --- | --- | --- | --- |
| permitAll | `GET /api/**` 광범위 허용 | **`/api/auth/signup`, `/api/auth/check-email` 만** | 광범위 허용은 나중에 실수로 민감 엔드포인트 노출 위험 |
| CORS origin | localhost:5173 + prod 2개 | localhost:4100 만 (개발) | 프론트 포트 다름. 프로덕션은 환경변수로 추후 |
| 로그인 필터 | 이 단계에서 함께 등록 | **다음 단계(로그인)** 에서 추가 | 단계별 커밋 용이성 |

## 주의
- `BCryptPasswordEncoder()` 기본 strength=10. 강화하려면 `new BCryptPasswordEncoder(12)` — 로그인 속도 체감 느려질 수 있음. 기본 유지.
- `SessionCreationPolicy.STATELESS` 는 JWT 전제. 회원가입 단계에서도 세션 만들 필요 없으니 먼저 걸어둠.
- CORS `allowCredentials=true` + 특정 origin 조합 필요 (와일드카드 불가). 지금은 쿠키 안 쓰지만 나중 대비.
