# 회원 가입 구현 — 개요

## 참고 프로젝트 리뷰 요약
`/Users/terecal/mapo-palantier-project` 의 회원가입/로그인 구현을 기반으로 한다. 원본은 **구조는 탄탄** 하지만 아래 문제들을 보정해서 복제한다.

### 그대로 가져올 것 (Solid)
- `Controller → Service → Repository → Domain` 계층 분리
- `User` 엔티티에 팩토리 메서드(DDD 스타일: `User.createNewUser(...)`)
- `GlobalExceptionHandler` + `ErrorCode` enum (일관된 에러 응답 envelope)
- `DuplicateEmailException` 같은 도메인 예외 → HTTP 상태 매핑
- `@Valid` + Bean Validation 으로 1차 검증
- 이메일 중복 체크 API 분리 (`GET /api/auth/check-email`)

### 바꿀 것
| 항목 | 원본 | 우리 |
| --- | --- | --- |
| 폼 라이브러리 | TanStack Form | **react-hook-form + zod** (이미 설치됨) |
| 비번 검증 비대칭 | FE만 complexity 강제 | **BE에도 `@Pattern` 동일하게** |
| 에러 메시지 | 전부 한국어 | 한국어 유지하되 상수/Enum 으로 모아두기 (나중에 i18n 쉽게) |
| permitAll 범위 | `GET /api/**` 광범위 | `/api/auth/{signup,login,check-email}` 만 |
| 하드코딩 테스트 계정 | LoginForm 에 기본값 | **제거** |
| 조직/권한 매핑 | 가입 시 조직별 권한 계산 | 스코프 밖 — 단순 `ROLE_USER` 만 부여 |
| 이메일 인증 | 없음 | **없음 유지** (기본 스코프) |

## 스코프
- **포함**: 이메일/비번 회원가입, 이메일 중복 체크, BCrypt 저장, 프론트 `/register` 페이지
- **제외**: 이메일 인증, 아이디/비번 찾기, SSO/OAuth2, 휴대폰 인증, 약관 동의 체크 UI (필드만 DB 에 `termsAgreedAt` 정도로)
