# Swagger 실행 방법

## 접속 URL

| 항목 | URL |
| --- | --- |
| Swagger UI | http://localhost:4101/swagger-ui/index.html |
| OpenAPI JSON | http://localhost:4101/v3/api-docs |
| OpenAPI YAML | http://localhost:4101/v3/api-docs.yaml |

> 백엔드가 **기동 중**이어야 접속 가능.

---

## 기본 흐름 (비인증 API 테스트)

1. 브라우저에서 `http://localhost:4101/swagger-ui/index.html` 접속
2. 원하는 API 클릭 → **Try it out** 클릭
3. Request body 입력 → **Execute**
4. Response 확인

**인증 없이 사용 가능한 엔드포인트:**
- `POST /api/auth/signup`
- `POST /api/auth/login`
- `POST /api/auth/refresh`
- `GET /api/auth/check-email`

## JWT 토큰 인증 흐름

### Step 1. 회원가입 (없으면)
```
POST /api/auth/signup
{
  "email": "test@example.com",
  "password": "Test123!@",
  "username": "테스터"
}
```

### Step 2. 로그인 → accessToken 복사
```
POST /api/auth/login
{
  "email": "test@example.com",
  "password": "Test123!@"
}
```
응답에서 `accessToken` 값 복사 (`eyJ...` 로 시작하는 문자열)

### Step 3. Swagger UI 에 토큰 등록
1. 우상단 **Authorize 🔒** 버튼 클릭
2. `bearerAuth` 항목의 **Value** 필드에 토큰 붙여넣기
   - `Bearer ` 접두사 **없이** 토큰만 입력 (`eyJ...`)
3. **Authorize** 클릭 → **Close**

### Step 4. 인증 필요한 API 테스트
이제 모든 API 요청에 자동으로 `Authorization: Bearer <token>` 헤더 포함.

예시:
```
GET /api/auth/me
→ 200 { "id": 1, "email": "test@example.com", "username": "테스터", "role": "ROLE_USER" }
```

## 토큰 만료 시

access token TTL = **30분**. 만료 후 API 호출 시 `401 AUTH_005`.

### Swagger 에서 refresh 방법
1. `POST /api/auth/refresh` 에 기존 `refreshToken` 입력
2. 응답의 새 `accessToken` 복사
3. 우상단 **Authorize** → 기존 토큰 지우고 새 토큰 붙여넣기

---

## 로그아웃
```
POST /api/auth/logout
```
- 인증 상태에서 호출 → DB 의 refresh token 삭제 → 204 응답
- 이후 refresh 시도 시 `401 AUTH_006`

## OpenAPI JSON 다운로드 (Postman 등 외부 도구 연동)

```bash
curl http://localhost:4101/v3/api-docs | python3 -m json.tool > api-docs.json
```

## 에러 코드 참고

| 코드 | HTTP | 의미 |
| --- | --- | --- |
| AUTH_001 | 409 | 이메일 중복 |
| AUTH_002 | 404 | 사용자 없음 |
| AUTH_003 | 401 | 이메일/비밀번호 불일치 |
| AUTH_004 | 403 | 비활성 계정 |
| AUTH_005 | 401 | 유효하지 않은 토큰 (만료 포함) |
| AUTH_006 | 401 | 유효하지 않은 리프레시 토큰 |
| COMMON_001 | 400 | 입력값 검증 실패 |
| COMMON_999 | 500 | 서버 오류 |
