# Swagger 설정 방법

## 사용 라이브러리
`springdoc-openapi` — Spring Boot 3/4 공식 지원, `/swagger-ui` + `/v3/api-docs` 자동 생성.

## 1. build.gradle 의존성

```gradle
implementation 'org.springdoc:springdoc-openapi-starter-webmvc-ui:2.8.8'
```

> 이미 추가됨: `twilio-callcenter-server/build.gradle`

---

## 2. SwaggerConfig.java

**경로:** `src/main/java/com/cj/twilio/callcenter/config/SwaggerConfig.java`

```java
@Configuration
public class SwaggerConfig {

    @Bean
    public OpenAPI openAPI() {
        String securitySchemeName = "bearerAuth";
        return new OpenAPI()
                .info(new Info()
                        .title("Twilio Callcenter API")
                        .description("Twilio Callcenter 백엔드 API 문서")
                        .version("v1.0.0"))
                .addSecurityItem(new SecurityRequirement().addList(securitySchemeName))
                .components(new Components()
                        .addSecuritySchemes(securitySchemeName, new SecurityScheme()
                                .name(securitySchemeName)
                                .type(SecurityScheme.Type.HTTP)
                                .scheme("bearer")
                                .bearerFormat("JWT")));
    }
}
```

**핵심 포인트:**
- `SecurityScheme` 에 `bearerAuth` 등록 → Swagger UI 우상단에 **Authorize** 버튼 생성
- `addSecurityItem` → 모든 API 에 기본으로 JWT 인증 적용
- 인증 불필요한 엔드포인트(`/api/auth/signup` 등)는 컨트롤러에 `@SecurityRequirements({})` 추가로 제외 가능 (현재는 미적용 — 테스트 편의상 전체 허용)

---

## 3. SecurityConfig 허용 경로 추가

**경로:** `src/main/java/com/cj/twilio/callcenter/config/SecurityConfig.java`

```java
.requestMatchers(
    "/swagger-ui/**",
    "/swagger-ui.html",
    "/v3/api-docs/**"
).permitAll()
```

이미 적용됨. Swagger UI 자체는 인증 없이 접근 가능, **API 호출은 토큰 필요**.

---

## 4. application.yaml (선택 옵션)

기본값으로 자동 동작하지만, 경로 커스터마이징이 필요하면 추가:

```yaml
springdoc:
  swagger-ui:
    path: /swagger-ui.html       # 기본: /swagger-ui/index.html
    tags-sorter: alpha            # 태그 알파벳 정렬
    operations-sorter: alpha      # 메서드 알파벳 정렬
  api-docs:
    path: /v3/api-docs            # 기본값과 동일
```

**현재 프로젝트엔 추가하지 않음** — 기본 경로로 충분.

## 5. 컨트롤러 어노테이션 (선택)

더 풍부한 문서가 필요하면 각 컨트롤러에 추가:

```java
@Tag(name = "Auth", description = "인증 관련 API")
@RestController
public class AuthController {

    @Operation(summary = "회원가입", description = "이메일/비밀번호/사용자명으로 계정 생성")
    @ApiResponses({
        @ApiResponse(responseCode = "201", description = "가입 성공"),
        @ApiResponse(responseCode = "400", description = "입력값 오류"),
        @ApiResponse(responseCode = "409", description = "이메일 중복")
    })
    @PostMapping("/signup")
    public ResponseEntity<SignupResponse> signup(...) { ... }
}
```

**현재 프로젝트엔 추가하지 않음** — 기본 자동 생성으로 충분.
