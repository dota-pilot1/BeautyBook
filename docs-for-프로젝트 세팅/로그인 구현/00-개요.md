# 로그인 구현 — 개요

## 전제
- `docs-for-프로젝트 세팅/회원 가입 구현/` 9개 파일이 전부 구현·검증된 상태.
- `users` 테이블에 BCrypt 해시로 저장된 테스트 계정 최소 1개 존재.

## 참고 프로젝트 리뷰 요약
`/Users/terecal/mapo-palantier-project` 의 로그인 구현은 JWT **듀얼 토큰(access + refresh)** 구조. 아래 보정 후 복제.

### 그대로 가져올 것
- JJWT 라이브러리 (`io.jsonwebtoken:jjwt-api:0.12.6`)
- Access + Refresh 듀얼 토큰 패턴
- `JwtTokenProvider` (토큰 생성/검증 전담)
- `JwtAuthenticationFilter` (`OncePerRequestFilter` 상속, Authorization 헤더 파싱)
- `type` 클레임("ACCESS"/"REFRESH")로 토큰 혼동 방지
- Refresh token DB 저장 (1 user = 1 token, 재로그인 시 덮어쓰기)
- 에러 envelope 은 회원가입 단계에서 만든 것 재사용
- `AuthService.login / refresh / logout`

### 바꿀 것
| 항목 | 원본(mapo) | 우리 |
| --- | --- | --- |
| JWT secret | `application.yml` 플레인스트링 | `.env` + `${JWT_SECRET}` 환경변수 바인딩 |
| 권한 로딩 | JWT 생성 시 `UserAuthorityRepository`, `OrganizationAuthorityMapper` 조회 | 그냥 `user.role` 하나만 |
| 토큰 저장 (프론트) | accessToken + refreshToken 모두 localStorage | **accessToken: 메모리 + localStorage** (새로고침 복원용), refreshToken: localStorage (편의상) — httpOnly 쿠키는 후속 과제로 |
| permitAll | `GET /api/**` 광범위 | 필요한 경로만 |
| 테스트 크리덴셜 | LoginForm 기본값에 박아둠 | 제거 |
| 폼 라이브러리 | TanStack Form | react-hook-form + zod |
| 토큰 회전 | 없음 (refresh 써도 기존 리프레시 재발행 안 함) | **회전** (refresh 요청마다 새 refresh 발급 + 기존 삭제) |

### 하지 않을 것 (스코프 밖)
- 이메일 인증
- 비번 재설정
- 소셜 로그인(OAuth2)
- Remember me / device 별 세션 관리
- Rate limiting (TODO 메모만)

## JWT 설계 고정값
| 항목 | 값 |
| --- | --- |
| 알고리즘 | HS256 (HMAC-SHA256) |
| Access token TTL | 30분 |
| Refresh token TTL | 7일 |
| Access claims | `sub=userId`, `email`, `role`, `type="ACCESS"` |
| Refresh claims | `sub=userId`, `type="REFRESH"` |
| Secret | 최소 32바이트 base64 문자열, `.env` 에서 주입 |
