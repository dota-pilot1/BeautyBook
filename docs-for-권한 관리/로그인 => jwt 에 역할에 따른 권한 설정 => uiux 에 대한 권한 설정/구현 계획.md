# 로그인 → JWT 권한 포함 → UI/UX 접근 제어 구현 계획

> 작성일: 2026-04-22

---

## 현재 상태

| 항목 | 현황 |
|------|------|
| JWT payload | `role` 코드 하나만 포함, `permissions` 없음 |
| login/me 응답 `UserSummary` | `role`만 포함, permissions 없음 |
| 프론트 `User` 타입 | `permissions` 필드 없음 |

---

## 1차 구현 (지금 할 것)

**목표**: 로그인 시 JWT + me 응답에 permissions 포함 → 프론트 프로필 페이지에서 내 권한 확인

### Step 1 — AuthService: permissions 조회 후 JWT에 전달

`role.getPermissions()` = `Role` 엔티티의 getter. `role_permissions` 테이블을 통해 연결된 `Permission` 목록을 반환한다 (`@ManyToMany LAZY`).

```java
// AuthService.issueTokens()
List<String> permCodes = user.getRole().getPermissions()
    .stream().map(Permission::getCode).toList();

String access = jwtTokenProvider.generateAccessToken(
    user.getId(), user.getEmail(), user.getUsername(),
    user.getRole().getCode(),
    permCodes   // ← 추가
);
```

### Step 2 — JwtTokenProvider: permissions 클레임 추가

```java
// 시그니처 변경
public String generateAccessToken(
    Long userId, String email, String username,
    String roleCode, List<String> permissions
) {
    return Jwts.builder()
        ...
        .claim("role", roleCode)
        .claim("permissions", permissions)   // ← 추가
        .compact();
}
```

### Step 3 — UserSummary: permissions 포함

```java
// Before
record UserSummary(Long id, String email, String username, RoleSummary role)

// After
record UserSummary(Long id, String email, String username, RoleSummary role, List<String> permissions)
```

→ login 응답과 `/api/auth/me` 응답 모두 자동 반영

### Step 4 — 프론트: User 타입에 permissions 추가

```ts
export type User = {
  id: number;
  email: string;
  username: string;
  role: RoleSummary;
  permissions: string[];   // ← 추가
};
```

### Step 5 — 프론트: 프로필 페이지에서 권한 확인

로그인한 유저의 `user.permissions` 목록을 프로필 페이지에서 표시.

## 2차 구현 (나중에 할 것)

**목표**: 백엔드 API + 프론트 UI를 permission 단위로 제어

- `JwtAuthenticationFilter` — JWT에서 permissions 파싱 → SecurityContext에 등록
- `UserPrincipal.getAuthorities()` — role + permissions 모두 반환
- `@PreAuthorize("hasAuthority('USER_VIEW')")` 로 API 잠금
- `usePermission("USER_DELETE")` 훅으로 버튼 조건부 렌더

---

## 주의사항

- `role.getPermissions()`는 LAZY 로딩이지만 `AuthService`가 `@Transactional`이라 별도 처리 불필요
- refresh 토큰 갱신 시에도 `issueTokens()` 공통 사용이므로 permissions 자동 포함
