## 현재 상태 (1차 완료 기준)

| 파일 | 현재 상태 |
|------|-----------|
| `JwtAuthenticationFilter` | JWT에서 `role`만 꺼냄, `permissions`는 버림 |
| `UserPrincipal.fromClaims()` | `permissions` 파라미터 없음 |
| `UserPrincipal.getAuthorities()` | `roleCode` 하나만 반환 → Spring Security는 permissions를 모름 |
| 각 Controller | `hasRole('ADMIN')` — role 단위만 |
| 프론트 UI 제어 | 없음 |

## 2차에서 할일
# 목표: 매 API 요청마다 JWT의 permissions를 Spring Security에 등록 → API를 permission 단위로 제어
# 방법: auth/security/JwtAuthenticationFilter 에 퍼미션도 넘기도록 설정 하기

---

## Step 1 (세트) — JwtAuthenticationFilter + UserPrincipal 수정

> Step 1과 Step 2는 세트다. JwtAuthenticationFilter에서 `fromClaims(..., permissions)`를 호출하려면
> UserPrincipal의 시그니처가 먼저 바뀌어 있어야 한다. 따로 배포하면 컴파일 에러.

### 1-A. UserPrincipal 수정

**파일**: `pilot-callcenter-server/src/main/java/com/cj/twilio/callcenter/auth/security/UserPrincipal.java`

```java
// 1. permissions 필드 추가
private final List<String> permissions;

// 2. 생성자에 permissions 추가
private UserPrincipal(Long id, String email, String username,
                      String passwordHash, String roleCode,
                      List<String> permissions, boolean active) { ... }

// 3. fromClaims() 시그니처 변경 — permissions 파라미터 추가
public static UserPrincipal fromClaims(
        Long id, String email, String username,
        String roleCode, List<String> permissions
) {
    return new UserPrincipal(id, email, username, null, roleCode, permissions, true);
}

// 4. getAuthorities() — role + permissions 모두 반환
//    Spring Security는 이 배열 하나로 hasRole(), hasAuthority() 둘 다 체크한다
@Override
public Collection<? extends GrantedAuthority> getAuthorities() {
    List<GrantedAuthority> authorities = new ArrayList<>();
    authorities.add(new SimpleGrantedAuthority(roleCode));
    permissions.forEach(p -> authorities.add(new SimpleGrantedAuthority(p)));
    return authorities;
}
```

완료 후 `getAuthorities()` 반환값:
```
[ROLE_ADMIN, USER_VIEW, USER_EDIT, USER_DELETE, ROLE_VIEW, ROLE_EDIT]
```

### 1-B. JwtAuthenticationFilter 수정

**파일**: `pilot-callcenter-server/src/main/java/com/cj/twilio/callcenter/auth/security/JwtAuthenticationFilter.java`

```java
// Before
UserPrincipal principal = UserPrincipal.fromClaims(
        jwtTokenProvider.getUserId(claims),
        jwtTokenProvider.getEmail(claims),
        jwtTokenProvider.getUsername(claims),
        jwtTokenProvider.getRole(claims)
);

// After — getPermissions()는 JwtTokenProvider에 이미 구현되어 있음
List<String> permissions = jwtTokenProvider.getPermissions(claims);
UserPrincipal principal = UserPrincipal.fromClaims(
        jwtTokenProvider.getUserId(claims),
        jwtTokenProvider.getEmail(claims),
        jwtTokenProvider.getUsername(claims),
        jwtTokenProvider.getRole(claims),
        permissions
);
```

**Step 1 완료 후 달라지는 것**: 모든 API 요청에서 Spring Security가 permissions를 인식한다.  
이 시점부터 컨트롤러에 `@PreAuthorize("hasAuthority('USER_VIEW')")` 달면 실제로 동작한다.

## Step 2 — 각 Controller에 @PreAuthorize 적용

**파일**: 각 Controller

```java
// Before
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/api/users")
public Page<UserListResponse> list(...) { ... }

// After
@PreAuthorize("hasAuthority('USER_VIEW')")
@GetMapping("/api/users")
public Page<UserListResponse> list(...) { ... }

@PreAuthorize("hasAuthority('USER_DELETE')")
@DeleteMapping("/api/users/{id}")
public void delete(...) { ... }
```

적용 대상:
| API | Permission |
|-----|-----------|
| GET /api/users | USER_VIEW |
| POST /api/users | USER_CREATE |
| DELETE /api/users/{id} | USER_DELETE |
| GET /api/roles | ROLE_VIEW |
| PUT /api/roles/{id}/permissions | ROLE_EDIT |

## Step 3 — 프론트: usePermission 훅 + 버튼 조건부 렌더

> 1차에서 이미 `authStore.user.permissions`까지 전달 완료.
> 추가 API 호출 없이 이미 저장된 데이터를 활용한다.

**파일**: `pilot-callcenter-front/src/shared/lib/usePermission.ts` (신규)

```ts
import { useAuth } from "@/entities/user/model/authStore";

export function usePermission(code: string): boolean {
  const { user } = useAuth();
  return user?.permissions.includes(code) ?? false;
}
```

**사용 예시** — 삭제 버튼:
```tsx
const canDelete = usePermission("USER_DELETE");

{canDelete && (
  <button onClick={() => setDeleteTarget(u)}>삭제</button>
)}
```
