## 목표

로그인 시 JWT payload에 role 외에 permissions 목록도 포함시키고,  
`/api/auth/me` 응답에도 permissions가 내려오도록 한다.

## 구현 내용 (완료)

### Step 1 — AuthService.issueTokens()에 permissions 추출 추가

**파일**: `pilot-callcenter-server/.../user/application/AuthService.java`

```java
private TokenResponse issueTokens(User user) {
    List<String> permCodes = user.getRole().getPermissions()
            .stream().map(p -> p.getCode()).toList();
    String access = jwtTokenProvider.generateAccessToken(
            user.getId(), user.getEmail(), user.getUsername(),
            user.getRole().getCode(), permCodes  // ← 추가
    );
    ...
}
```

- `user.getRole()` → `EAGER` 로딩이라 별도 쿼리 없음
- `getPermissions()` → `LAZY`이지만 `@Transactional` 내부라 안전하게 접근
- 로그인 / refresh 토큰 재발급 / 모든 `issueTokens()` 경로에 자동 적용

---

### Step 2 — JwtTokenProvider.generateAccessToken()에 permissions 클레임 추가

**파일**: `pilot-callcenter-server/.../auth/jwt/JwtTokenProvider.java`

```java
// 시그니처에 List<String> permissions 추가
public String generateAccessToken(
        Long userId, String email, String username,
        String roleCode, List<String> permissions
) {
    return Jwts.builder()
            ...
            .claim("role", roleCode)
            .claim("permissions", permissions)  // ← 추가
            .compact();
}

// 파싱 메서드 추가
@SuppressWarnings("unchecked")
public List<String> getPermissions(Claims c) {
    Object raw = c.get("permissions");
    if (raw instanceof List<?> list) return (List<String>) list;
    return List.of();
}
```

JWT payload 결과:
```json
{
  "role": "ROLE_ADMIN",
  "permissions": ["USER_VIEW", "USER_EDIT", "USER_DELETE", "ROLE_VIEW", "ROLE_EDIT"]
}
```

### Step 3 — UserSummary에 permissions 필드 추가

**파일**: `pilot-callcenter-server/.../user/presentation/dto/UserSummary.java`

```java
public record UserSummary(
        Long id, String email, String username,
        RoleSummary role,
        List<String> permissions  // ← 추가
) {
    public static UserSummary from(User u) {
        List<String> permCodes = u.getRole().getPermissions()
                .stream().map(p -> p.getCode()).toList();
        return new UserSummary(u.getId(), u.getEmail(), u.getUsername(),
                RoleSummary.from(u.getRole()), permCodes);
    }
}
```

- `POST /api/auth/login` 응답 자동 반영
- `GET /api/auth/me` 응답 자동 반영

## 데이터 흐름 요약

```
로그인 요청
  ↓
AuthService.issueTokens(user)
  → user.getRole().getPermissions() 로 permCodes 추출
  → JwtTokenProvider.generateAccessToken(..., permCodes)
      → JWT payload: { role, permissions: [...] }
  → UserSummary.from(user)
      → 응답 JSON: { user: { role, permissions: [...] } }
  ↓
프론트 authStore.user.permissions 에 저장
  ↓
/profile 페이지에서 user.permissions 목록 렌더링
```

---

## 주의사항

- `User.role` → `EAGER`, `Role.permissions` → `LAZY`: 쿼리 2번 발생 (N+1 아님 — 단일 유저)
- refresh 토큰 재발급 시에도 `issueTokens()` 공통 사용이므로 permissions 자동 갱신
- permissions는 로그인 시점의 스냅샷이므로, 역할의 권한이 변경되면 재로그인 필요
