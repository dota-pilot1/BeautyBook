# 스텝 11: i18n / 아이콘 / 외부 링크

> DB 와 관리 UI 에는 필드가 모두 있으나, **헤더 렌더 단계에서 i18n / 아이콘을 아직 소비하지 않는다.** 이 문서는 현재 상태와 미완료 항목을 정확히 기록한다.

---

## 1. i18n — `labelKey` + `label` 폴백

### 현재 상태

- 시더가 `labelKey` 를 채워 두었다: `nav.dashboard`, `nav.admin`, `nav.users`, `nav.rolePermissions`, `nav.siteSettings`, `nav.menuManagement`.
- **헤더는 `item.label` 을 그대로 사용** — `labelKey` 해석 없음.
- locale json 파일에 nav 키가 있어도 현재는 미적용.

### 도입 시

헤더 `NavItem` / `DropdownMenu` 에서:
```tsx
import { useTranslation } from "react-i18next";

const { t } = useTranslation("nav");
const label = item.labelKey
  ? t(item.labelKey, { defaultValue: item.label })
  : item.label;
```

### 키 네이밍 규약 (현재 시더 기준)
- 점 표기법, camelCase: `nav.dashboard`, `nav.rolePermissions`, `nav.menuManagement`.
- 정규식 검증 `^nav\.[a-z][a-zA-Z0-9]*$` 권장.

### locale 파일
`public/locales/{ko,en}/nav.json` 에 매핑 필요. 현재 일부만 존재할 가능성 있음 — 도입 시 누락 키 추가.

## 2. 아이콘 — `lucide-react` 문자열 매핑

### 현재 상태

- 시더 값: `LayoutDashboard`, `Settings`, `Users`, `ShieldCheck`, `Menu`.
- **헤더에 아이콘 렌더 자체가 없다.**
- 관리 UI `DetailPanel` / `MenuFormDialog` 는 `<input>` 자유 입력. autocomplete 없음.

### 도입 시

```tsx
import * as Lucide from "lucide-react";

function MenuIcon({ name, className }: { name: string | null; className?: string }) {
  if (!name) return null;
  const Cmp = (Lucide as any)[name];
  if (typeof Cmp !== "function") return null;
  return <Cmp className={className ?? "h-3.5 w-3.5"} />;
}
```

### 번들 최적화

`import * as Lucide` 는 tree-shaking 과 친하지 않다. v2 에서:
- `shared/ui/icons/menuIcons.ts` 에 20~30개 whitelist 만 둠.
- 관리 UI Select 도 이 whitelist 에서만 선택.

## 3. 외부 링크 — `isExternal`

### 실제 구현 (DropdownMenu 의 자식만)

```tsx
<Link
  href={child.path ?? "#"}
  target={child.isExternal ? "_blank" : undefined}
  rel={child.isExternal ? "noopener noreferrer" : undefined}
>
  {child.label}
</Link>
```

- `isExternal=true` 면 `target="_blank"` + `rel="noopener noreferrer"` 자동.
- **1차 메뉴 `NavLink`(자식 없음) 에서는 isExternal 처리 없음** → 필요 시 추가.
- 시각적 ↗ 아이콘 **미표시** (lucide 아이콘 렌더 자체를 안 해서).

### 관리 UI 보안 검증

- 현재는 `path` 를 그대로 저장. URL 파싱 검증 없음.
- 도입 시: `new URL(path)` 로 프로토콜 `http(s):` 체크.

---

## 4. 폴더형 메뉴 (`path=NULL`)

### 실제 동작

- 시더의 `ADMIN` 이 `path=null` 폴더형.
- `buildTree` 로 만든 `MenuItem` 의 `children.length > 0` → `DropdownMenu` 로 렌더.
- `children.length === 0` 인 폴더형이 생기면 `<NavLink href="#">` 로 렌더되는 버그 가능성 있음 (현재는 시드에 발생 케이스 없음).

### 방어

도입 시 헤더 `buildTree` 끝에:
```ts
const pruneEmptyFolders = (items: MenuItem[]): MenuItem[] =>
  items
    .map(i => ({ ...i, children: pruneEmptyFolders(i.children) }))
    .filter(i => i.path != null || i.children.length > 0);
```

---

## 5. 유지보수 팁

- `labelKey` 가 규약을 어기는 row 검사: `SELECT * FROM menus WHERE label_key NOT REGEXP '^nav\\.[a-z][a-zA-Z0-9]*$'`
- `icon` 값이 lucide 에 없는 이름: 콘솔에 `console.warn(\`icon '\${name}' not found\`)` 출력 고려.
- `isExternal=true` + `path` 가 `http` 로 시작 안 하면 관리 UI 에서 경고 배너.

---

## 결론 — 현재 우선순위

이 3개 필드는 **선택**(optional) 이고, 현재도 시스템은 정상 동작한다.

1. **i18n 적용** (도입 비용: 낮음, 가치: 중간) — 다국어 지원 언제 추가할지에 따라.
2. **아이콘 렌더** (도입 비용: 낮음, 가치: 시각적) — 헤더 UX 개선 필요시.
3. 1차 메뉴 외부 링크, 폴더 방어 — 해당 케이스 생길 때.
