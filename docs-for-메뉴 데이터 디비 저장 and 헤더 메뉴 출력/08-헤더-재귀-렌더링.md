# 스텝 8: 헤더 재구성 — DB 기반 동적 렌더링

> 기존 [Header.tsx](../../beauty-book--front/src/widgets/header/ui/Header.tsx) 의 `AdminDropdown` 하드코딩 구조를 제거하고 **`buildTree(flatMenus, userRole)` → `NavItem` / `DropdownMenu` 로 동적 렌더** 로 바꿨다.

---

## 실제 컴포넌트 구조

원래 계획은 단일 재귀 `<MenuNode>` 로 N차 전부 처리였지만, **실제는 2단계 전용** 으로 단순화했다:

```
<Header>
  ├ <NavLink>  (자식 없는 1차)
  └ <DropdownMenu>  (자식 있는 1차)
       └ <Link> × N   (2차, flat)
```

```ts
function NavItem({ item }: { item: MenuItem }) {
  if (item.children.length > 0) {
    return <DropdownMenu item={item} />;
  }
  return (
    <NavLink href={item.path ?? "#"} exact={item.path === "/dashboard"}>
      {item.label}
    </NavLink>
  );
}
```

- **3차 이상 미지원** — `DropdownMenu` 가 `item.children.map(child => <Link>)` 로 평탄하게 렌더. 자식의 자식이 있으면 표시 안 됨.
- 현재 시더에 3차 메뉴가 없어서 문제 없음. 필요 시 `DropdownMenu` 의 자식 렌더를 재귀로 바꿔야 함.

## Header 최상위 로직 (실제)

```tsx
export function Header() {
  const { status, user } = useAuth();
  const userRole = user?.role?.code ?? null;

  const { data: flatMenus = [] } = useQuery({
    queryKey: ["menus"],
    queryFn: menuApi.getAll,
    staleTime: 1000 * 60 * 5,
  });

  const tree = buildTree(flatMenus, userRole);   // 필터 포함

  return (
    <header className="sticky top-0 z-50 ...">
      <div className="flex h-14 ...">
        <nav className="flex items-center gap-6">
          <Link href="/">BeautyBook</Link>
          {status === "authenticated" &&
            tree.map((item) => <NavItem key={item.id} item={item} />)}
        </nav>
        <div className="flex items-center gap-2">
          <LanguageSelect />
          <ThemeSwitcher />
          {status === "authenticated" ? <>...</> : <>로그인/회원가입</>}
        </div>
      </div>
    </header>
  );
}
```

- 비로그인 상태에서는 메뉴 영역이 비고 로그인/회원가입 버튼만 표시.
- `UserDropdown`, `ThemeSwitcher`, `LanguageSelect`, `LogoutButton` 은 **기존 그대로 유지** (메뉴 DB화 범위 밖).

## `DropdownMenu` 구현

```tsx
function DropdownMenu({ item }: { item: MenuItem }) {
  const [open, setOpen] = useState(false);
  const ref = useRef<HTMLDivElement>(null);
  const pathname = usePathname();

  // active: 자식 중 현재 경로를 포함하는 게 있으면 active
  const isActive = item.children.some(
    (c) => c.path && pathname.startsWith(c.path)
  );

  // 바깥 클릭으로 닫기
  useEffect(() => {
    const handler = (e: MouseEvent) => {
      if (ref.current && !ref.current.contains(e.target as Node)) setOpen(false);
    };
    document.addEventListener("mousedown", handler);
    return () => document.removeEventListener("mousedown", handler);
  }, []);

  return (
    <div ref={ref} className="relative">
      <button onClick={() => setOpen(v => !v)} className={isActive ? ... : ...}>
        {item.label}
        <Chevron />
      </button>
      {open && (
        <div className="absolute left-0 top-full mt-2 ...">
          {item.children.map((child) => (
            <Link
              key={child.id}
              href={child.path ?? "#"}
              target={child.isExternal ? "_blank" : undefined}
              rel={child.isExternal ? "noopener noreferrer" : undefined}
              onClick={() => setOpen(false)}
            >
              {child.label}
            </Link>
          ))}
        </div>
      )}
    </div>
  );
}
```

## Active 판정

```ts
const isActive = item.children.some(
  (c) => c.path && pathname.startsWith(c.path)
);
```

- **1-depth 만 검사** — 자식의 자식은 보지 않음. 3차 메뉴 생기면 재귀 `collectPaths` 로 바꿔야 함.
- 기존 하드코딩 `pathname.startsWith("/users") || startsWith("/role-permissions") || ...` 삭제 완료. 트리 구조 자체가 active 규칙.

## i18n / 아이콘 / 외부 링크 — 현재 상태

| 기능 | 구현 상태 |
|------|:--------:|
| `labelKey` → `t(labelKey, { defaultValue: label })` 해석 | ❌ **미구현**. 현재는 `item.label` 그대로 사용 (한국어) |
| `icon` 문자열 → lucide 컴포넌트 | ❌ **미구현**. 헤더에 아이콘 표시 자체를 안 함 |
| `isExternal` → `target="_blank"` | ✅ 구현 (DropdownMenu 의 자식만) |

> 시더는 `labelKey`, `icon` 을 모두 채워 두었으나 헤더에서 소비하지 않는다. 관리 UI 에서는 편집 가능. 소비는 v2.

## 계획 대비 결정 로그

| 계획 | 실제 | 이유 |
|------|------|------|
| 단일 재귀 `<MenuNode depth={}>` | `NavItem` + `DropdownMenu` 2단계 | N차 메뉴가 없어서 재귀 필요성 없음. 단순함 우선 |
| hover 로 서브 드롭다운 오픈 | 클릭 토글 | 기존 Header UX 유지 |
| `aria-haspopup="menu"` 등 접근성 속성 | 일부만 | v2 |
| `filterVisibleForUser` 유틸 | Header 의 `buildTree` 안에 인라인 | 중복 없음 |
| `pruneEmptyFolders` | 미적용 | 현재 빈 폴더 케이스 없음. `requiredRole` 로 `ADMIN` 이 필터되면 자식도 자동 드롭됨 |
| `labelKey` → i18n 해석 | 미구현 | 우선순위 낮음 |

---

## 남은 작업 (향후)

1. **N차 메뉴 지원** — `DropdownMenu` 를 재귀 컴포넌트로 교체.
2. **i18n 해석** — `const label = item.labelKey ? t(item.labelKey, { defaultValue: item.label }) : item.label;`
3. **아이콘 렌더** — `import * as Lucide from "lucide-react"` → `Lucide[item.icon]` 매핑 ([11](./11-i18n-아이콘-외부링크.md)).
4. **빈 폴더 방어** — 자식이 전부 필터링된 드롭다운은 1차 레일에서 제거.
5. **접근성** — `aria-expanded`, 키보드 탐색(Esc/Arrow).
