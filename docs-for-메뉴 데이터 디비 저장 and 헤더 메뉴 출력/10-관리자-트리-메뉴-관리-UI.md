## 드래그 앤 드롭 (DnD)

라이브러리: `@dnd-kit/core` + `@dnd-kit/sortable` + `@dnd-kit/modifiers` + `@dnd-kit/utilities`.

### 허용 연산 (현재)

| 제스처 | 결과 | 구현 |
|--------|------|:----:|
| **같은 부모 안 위/아래 이동** | 형제 순서 변경 | ✅ |
| 다른 부모로 드롭 | — | ❌ (부모 변경은 DetailPanel 의 "부모 메뉴" Select 로만) |
| 자기 후손 위로 드롭 | — | 불가능 (다른 부모 DnD 자체 없음) |

### 구현 (요약)

```tsx
<DndContext
  sensors={sensors}
  collisionDetection={closestCenter}
  onDragEnd={handleDragEnd}
  modifiers={[restrictToVerticalAxis, restrictToWindowEdges]}
>
  <SortableContext items={tree.map(i => i.id)} strategy={verticalListSortingStrategy}>
    {tree.map(item => <TreeNode ... />)}    {/* 재귀 */}
  </SortableContext>
</DndContext>
```

- `TreeNode` 가 재귀적으로 자신의 자식을 `<SortableContext>` 로 감쌈 → **depth 별로 각자 소트**.
- 드래그 시작: `@dnd-kit` 의 `useSortable({ id: item.id })`.
- `restrictToVerticalAxis`, `restrictToWindowEdges` 로 드래그 제약.

### `handleDragEnd` 로직

```ts
const handleDragEnd = (event: DragEndEvent) => {
  const { active, over } = event;
  if (!over || active.id === over.id) return;

  const activeItem = flat.find(m => m.id === active.id);
  const parentId = activeItem.parentId;
  const siblings = flat
    .filter(m => m.parentId === parentId)
    .sort((a, b) => a.displayOrder - b.displayOrder);

  const oldIdx = siblings.findIndex(m => m.id === active.id);
  const newIdx = siblings.findIndex(m => m.id === over.id);
  if (oldIdx === -1 || newIdx === -1) return;

  const reordered = arrayMove(siblings, oldIdx, newIdx).map((m, i) => ({
    ...m, displayOrder: i,
  }));

  setLocalFlat(/* flat 에서 reordered 로 치환 */);
  saveMutation.mutate(reordered);    // Promise.all(개별 PATCH)
};
```

- 같은 부모 안에서만 순서 바꿈. `over` 가 다른 부모의 형제면 `findIndex` 가 -1 → 무시.
- `displayOrder` 를 **0부터 연속값** 으로 재할당.
- 낙관적 local 반영 + 서버 저장 병행.

## DetailPanel (편집 + 삭제)

```tsx
function DetailPanel({ menu, allMenus, onSaved, onDeleted }) {
  const [form, setForm] = useState<UpdateMenuBody>(toUpdateBody(menu));

  const mutation = useMutation({
    mutationFn: () => menuApi.update(menu.id, form),
    onSuccess: () => { toast.success("저장되었습니다."); qc.invalidateQueries(["menus"]); onSaved(); },
  });
  const deleteMutation = useMutation({
    mutationFn: () => menuApi.delete(menu.id),
    onSuccess: () => { toast.success("삭제되었습니다."); qc.invalidateQueries(["menus"]); onDeleted(); },
  });

  // 부모 선택: 루트 메뉴만 선택 가능 (자신 제외)
  const roots = allMenus.filter(m => m.parentId === null && m.id !== menu.id);

  return (/* 필드들 + [삭제] [저장] */);
}
```

### 필드
`부모 메뉴` / `레이블` / `i18n 키` / `경로` / `아이콘` / `필요 역할` / `표시 순서` / `표시` / `외부 링크`.

### 부모 선택 제약
**루트 메뉴만 부모로 선택 가능** (`m.parentId === null`). 깊은 트리 이동 금지 = 순환 참조 발생 여지 없음.

### 삭제
`window.confirm` 으로 간단 확인. 자식 있으면 서버 측 FK 위반 → 에러 toast.

## 생성 다이얼로그 ([MenuFormDialog.tsx](../../beauty-book--front/src/features/menu-management/MenuFormDialog.tsx))

- 상단 `+ 추가` 버튼 클릭 → 모달 오픈.
- react-hook-form 사용. `code` 필드는 생성 시에만 편집 가능.
- 부모 선택도 **루트만**.
- 성공 시 `["menus"]` invalidate → 트리에 즉시 반영.

## 쿼리 상태 관리

```ts
const [localFlat, setLocalFlat] = useState<MenuRecord[] | null>(null);
const { data: serverMenus = [] } = useQuery({ queryKey: ["menus"], queryFn: menuApi.getAll });
const flat = localFlat ?? serverMenus;
```

- `localFlat` 은 DnD 직후의 낙관적 상태 (서버 왕복 전 즉시 반영).
- `invalidateQueries` 가 `serverMenus` 를 새로 받아오면 자동으로 최신.
- **Optimistic update with rollback 은 미구현** — 서버 실패 시 toast 만 뜨고 local 이 틀린 상태로 남음 (이후 invalidate 로 복구됨).

## 에러 처리

- `MENU_CODE_DUPLICATE` → 다이얼로그에서 toast.
- `MENU_PARENT_NOT_FOUND` → 거의 발생 안 함 (선택지가 실제 존재하는 것들).
- 네트워크/500 → 공통 `toastError`.
- 접근성 / 키보드 DnD / 미리보기는 v2.
