# AI 개발 방법론 — 정답 자판기 문제와 해결 전략

작성일: 2026-04-23
목적: AI 보조 개발에서 발생하는 구조적 문제를 통제하고, 실무에서 안정적으로 활용하기 위한 기준 정립

---

## 1. AI의 본질

AI는 정답 생성기가 아니라 "입력 증폭기(Input Amplifier)"

- 좋은 인풋 → 좋은 결과 증폭
- 잘못된 인풋 → 완성도 높은 오답 생성

---

## 2. 핵심 문제

### [1] 인풋 의존성

- 아키텍처 이해 부족 → 구조 붕괴
- DB 설계 부족 → 데이터 무결성 문제
- 도메인 이해 부족 → 비즈니스 로직 오류
- 디자인 감각 / UX 흐름 이해 부족 → UI/UX 오류
- 계획 부족 → 반복 수정 발생

=> 결과: 방향 자체가 틀어진 상태에서 고품질 오답 생성

### [2] 블랙박스 구현

- 내부 동작 이해 없이 결과만 사용

=> 결과:
- 장애 원인 분석 불가
- 요구사항 변경 대응 불가
- 확장 시 구조 붕괴

=> 핵심: "돌아가는 코드" ≠ "이해하는 코드"

### [3] 속도 중심 개발

- 빠른 구현에 집중하면서 구조 무시

=> 결과:
- 레이어 분리 붕괴
- 네이밍 컨벤션 붕괴
- 중복 코드 증가
- 테스트 부재

=> 결론: 초기 속도 ↑ / 유지보수 비용 폭증

---

## 3. 해결 전략

### 핵심 원칙

AI는 생성 도구가 아니라 "확장 도구"

### 실행 방법

**[1] 보일러플레이트 / 프로토타입 선 구축**

직접 이해하고 검증한 구조를 템플릿화

예시:
- 인증/보안: JWT Rotate, Spring Security
- 권한: User → Role → Permission (RBAC)
- 에러 처리: ErrorCode + GlobalExceptionHandler
- 메뉴: DB 기반 트리 + 권한 게이팅
- 배포: Docker, Nginx, CI/CD
- 프론트: FSD + 상태관리 패턴

=> 핵심: 반복 구조는 AI가 아니라 "내가 먼저 검증"

**[2] AI 역할 제한**

❌ 전체 시스템 생성 요청
예: "회원 관리 시스템 만들어줘"

✅ 구조 기반 확장 요청
예: "이 User 구조 기반으로 프로필 수정 API 추가해줘"

=> AI는 확장만 담당

**[3] 아웃풋 검증**

AI 결과는 반드시 검증 후 사용

체크 기준:
- 레이어 구조 위반 여부
- 도메인 로직 위치 적절성
- 코드 설명 가능 여부

=> 설명 못하면 이해 못한 코드 → 사용 금지

---

## 4. AI 시대의 개발 실력 재정의

과거: 타자 속도, 코드 암기량, 구현 속도

현재:
- 인풋 설계 능력 (무엇을 요청할지 정확히 정의)
- 단위 분해 능력 (기능을 얼마나 잘게, 명확하게 쪼개는가)
- 아웃풋 검증 능력 (AI 결과가 맞는지 판단)

사람 = 설계 + 검증
AI = 구현 + 확장

---

## 5. AI 시대 개발 프로세스

```
Before (과거)
설계 → 개발 → 테스트

Now (AI 시대)
설계 (훨씬 중요해짐)
↓
AI로 빠르게 구현
↓
검증 루프 (필수)
↓
반복 개선
```

개발 = 구현 싸움 ❌
개발 = 설계 + 검증 싸움 ⭕

AI 개발의 본질은 속도가 아니라

**"검증된 구조 위에서의 제어된 확장"**

## 부록. "AI 한계론"에 대한 정리

### 1. 그 문제들은 원래 개발자가 책임지는 영역이다

- 사이드 이펙트 통제
- 상태 흐름 설계
- 시스템 일관성 유지
- 예외 상황 처리
- 검증과 테스트

"AI가 못하네"라고만 말하는 건 프레임이 틀렸다. 원래 그건 개발자 역할이다.

### 2. AI 한계를 말할 때 자주 프레임이 틀어진다

- "AI가 큰 시스템 일관성을 못 맞춘다"
- "AI가 복잡한 상태를 완벽 보장 못한다"
- "AI가 사이드 이펙트를 놓친다"

이 말 자체는 틀리지 않지만, AI가 혼자 완성품을 만들어야 하는 기준처럼 말하면 이상해진다.
AI를 개발 보조가 아니라 자동 조제기처럼 보는 착각에 가깝다.

### 3. 정확한 표현

> AI의 한계가 아니라, AI를 검토자 없는 단독 설계자/구현자로 쓰면 위험하다.

AI가 부족해서 문제라기보다,
사람이 책임져야 할 판단을 AI에 넘겨버린 사용 방식이 문제인 경우가 훨씬 많다.
